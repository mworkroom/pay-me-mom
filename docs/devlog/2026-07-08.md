# 2026-07-08 Pay Me Mom Development Log

## Work Summary

아이폰 화면에서 카드 내부 요소가 예상보다 좁은 폭에 배치되면서 월 표시, 정산 버튼, 월별 합계, 항목 액션 버튼, 날짜 입력창이 줄바꿈되거나 화면 밖으로 밀리는 문제를 정리했다. 460px 기준은 유지하되 실제 모바일 가용 폭 안에서 주요 정보가 한 줄에 유지되도록 CSS 레이아웃을 조정했다.

## Work Completed

* 로컬 개발 서버에서는 `/` 경로로 열리고, 빌드 결과는 GitHub Pages용 `/pay-me-mom/` 경로를 유지하도록 Vite `base` 설정을 실행 모드별로 분리했다.
* 상단 요약 카드가 420px 이하에서 세로로 바뀌지 않도록 하고, 금액과 월 배지가 한 줄에 들어가도록 간격과 글자 크기를 줄였다.
* 월별 목록 헤더의 월 라벨과 `건수 · 금액` 요약을 같은 줄에서 좌우 정렬하도록 변경했다.
* 지출 항목 카드에서 날짜, 제목, 금액, 수정/삭제 버튼이 한 줄에 표시되도록 카드 그리드를 재구성했다.
* 모바일 Safari에서 날짜 입력창이 부모 폭을 넘지 않도록 폼 라벨과 입력 요소에 `min-width: 0`을 적용하고 가로 오버플로를 차단했다.
* iOS date input이 기본 컨트롤의 최소 폭을 고집하는 문제를 줄이기 위해 `input[type="date"]`에 `appearance: none`, `inline-size`, `max-inline-size`, WebKit date value 정렬 규칙을 추가했다.
* `npm install` 뒤 생성된 `node_modules`와 빌드 결과물 `dist`가 커밋 대상에 섞이지 않도록 `.gitignore`에 제외 규칙을 추가했다.
* 별도의 기능 변경 없이 반응형 CSS만 조정했다.

## Decisions

* GitHub Pages 배포 경로와 로컬 개발 경로를 같은 값으로 묶어두면 VS Code에서 `npm run dev`로 확인할 때 주소가 헷갈릴 수 있다고 판단했다. 배포 빌드에서는 기존 경로를 유지하고 개발 서버에서만 루트 경로를 쓰도록 분리했다.
* J님이 맞춘 460px 기준은 유지했다. 문제의 핵심은 최대 앱 폭보다 카드 내부 패딩과 모바일 미디어 쿼리의 세로 전환이 실제 사용 가능 폭을 줄이는 데 있었기 때문이다.
* 항목 카드는 버튼을 아래로 내리는 대신 카드 자체를 2열 그리드로 바꿨다. 반복 목록에서는 한 행에서 날짜, 내용, 금액, 액션을 훑는 편이 모바일에서도 더 빠르게 읽힌다고 판단했다.
* 월별 헤더는 JSX 구조를 바꾸지 않고 CSS만으로 내부 `div`를 좌우 정렬했다. 현재 데이터 구조와 렌더링 로직을 건드리지 않아 변경 범위를 줄일 수 있었다.
* 의존성 본문은 `package-lock.json`으로 재현할 수 있으므로 `node_modules`는 저장소에 포함하지 않기로 했다. 빌드 결과물 역시 소스에서 다시 만들 수 있어 커밋하지 않는 편이 변경 이력을 작게 유지한다.
* 날짜 입력창은 일반 input 규칙만으로 iOS Safari의 내부 date 컨트롤 폭을 제어하기 어렵다고 보고, date 타입 전용 규칙을 별도로 두었다.

## Current Status

* 로컬 개발 서버는 `http://localhost:5173/`로 접근할 수 있도록 설정했다.
* 모바일 폭에서 요청된 네 가지 레이아웃 문제를 CSS로 수정했다.
* Git 커밋 대상에서 의존성 설치 결과물과 빌드 산출물이 제외되도록 정리했다.
* iOS 날짜 입력창이 부모 카드 폭 안에 머물도록 date 타입 전용 폭 제약을 추가했다.
* 텍스트가 긴 항목 제목은 한 줄 유지 조건을 위해 말줄임 처리된다.

## Next Steps

* 금액 자릿수가 더 길어지는 경우 버튼과 금액 사이의 여백이 충분한지 확인한다.
* iOS date input이 여전히 튀어나오면 native date input을 숨기고 텍스트형 날짜 입력 또는 커스텀 선택 UI로 바꾸는 방안을 검토한다.
